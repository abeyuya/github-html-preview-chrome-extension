長尺コンテンツのサンプル

このページはビューポートより十分に背の高い長文コンテンツです。プレビューの 高さ追従（overlay がビューポートを超えて伸びること）と、 ページ内アンカーによるスクロール挙動を確認できます。外部リソースには依存せず、 CSS / JS はすべてインラインです。

1. はじめに

長いドキュメントでは、プレビュー領域が内容の高さに追従して伸びる必要があります。 固定高でクリップされると末尾が読めなくなってしまうため、内側 iframe が報告する コンテンツ高さに合わせて overlay の高さが更新されることを、このページで確認します。

ポイント: コンテンツがビューポートより高い場合、overlay の高さは window.innerHeight を超えて設定されるはずです。短いページ（basic など）とは挙動が異なります。

2. 背景

GitHub の blob ページは HTML をソースとして表示するため、ブラウザ上で レンダリング結果を直接確認できません。本拡張機能は、ソース表示を差し替えて 隔離された iframe 内でレンダリングし、ローカルでのプレビューを可能にします。

長文・多セクションのページは、見出しジャンプ・スクロール量・高さ追従など、 短いサンプルでは表面化しない挙動を露出させます。

6. 高さ計測

内側 iframe は scrollHeight を計測して親へ通知し、overlay がその高さに 追従します。ビューポートを超える長文では、overlay の高さが大きく増えることを 確認してください。

フィードバックループ防止のため、純粋な高さ変化は無視され、幅変化・DOM 変化・ サブリソース読み込み時にのみ再計測されます。長文でも末尾まで表示されるのが正しい挙動です。

8. セキュリティ

プレビュー対象 HTML は allow-same-origin を持たない隔離 iframe で実行され、 GitHub のセッションやクッキーには到達できません。任意の HTML / JS を安全に実行するための 境界です。

この隔離により、親ページ（github.com）から内側 DOM を読むことはできません。動作確認では スクリーンショットと overlay の高さで描画を判断します。

9. 既知の制限

<link rel="stylesheet"> や <script src> で読み込む 相対パスの CSS / JS は、raw が text/plain + nosniff で返すため ブラウザが適用を拒否します。スタイルや動的処理はインラインにするか、CDN の絶対 URL を 利用してください。

また、短いコンテンツのページではプレビュー領域が既定のペイン高さで頭打ちになり、 下部に余白が出ることがあります。

10. トラブルシュート

プレビューが表示されない場合は、拡張機能が最新ビルドで読み込まれているか、対象が .html / .htm の blob ページかを確認します。ソース変更後は npm run build と拡張機能の再読み込みが必要です。

11. FAQ

Q. なぜ三段ネストの iframe なのか？
A. permissive な CSP を持つ sandbox ページと、隔離実行用の inner iframe を分けることで、 安全性と機能性を両立するためです。

Q. 高さが伸びないのはバグ？
A. ビューポートより短いページでは既定ペイン高さで頭打ちになります。長文の本ページで 高さが伸びていれば追従機能は正常です。
